iNaturalist vision saliency

Runs entirely in your browser with ONNX Runtime Web. The network is the same vision weights as the official v25.01.15 release (INatVision_Small_2_fact256_8bit.tflite), exported here to a dequantized ONNX graph so we can run many forward passes for a saliency estimate. Preprocessing matches the mobile model: 299×299, RGB 0–255.


The page starts with a sample bear image. Choose another photo to replace it.


Each sample adds Gaussian noise to the input (scaled by σ below), runs one forward pass, and contributes to an average that approximates how sensitive the predicted class probability is to each pixel (a SmoothGrad-style estimate without backprop). More samples reduce variance in the heatmap but linearly increase runtime.


Standard deviation of the noise, in the same units as pixel values (0–255 per RGB channel), before values are clipped. Larger σ blurs the signal over a wider neighborhood of the image (smoother, more global sensitivity); smaller σ stays closer to the behavior of a sharp local gradient but can look noisier with few samples.

Technical note

The Python tool in tools/inat_vision_saliency/ uses PyTorch autograd for an exact gradient of the softmax probability. This page uses only forward passes and averages random-direction finite differences (Monte Carlo), so maps can differ slightly while remaining interpretable.

Default photo: bear image from PlaceBear (saved in-repo for this demo).
